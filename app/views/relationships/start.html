Some prototype mock ups for how we could handle Relationships.

Option 1 - Freetext

A freetext box for users to type in anything they want for their answer.

Users might not know what to put into the text box.

View freetext prototype

Option 2 - Select / Dropdown

Dropdowns that present all options to the user.

Advised as a last resort, but better than a freetext box?

View dropdown prototype

Option 3 - Asking questions

Group relationship items into categories, and let users drill down to select their answer.

When presented with a long dropdown (as in option 2) - GDS advises this route - asking questions.

View questions prototype

Option 4 - Radios

A long list of radio options

Could possibly be too long a list to present to users? Screenreader or accessibility concerns? Could users get trapped in the long list?

View radios prototype

Option 4.1 - Radios with category headings

Option 5 - Progressive Radios

Present users with 5 radios options with a 'show more' button.

Accessibility concerns over hiding content behind progressive disclosure?

View progressive radios prototype

Present users with a text input. They start typing and we autosuggest answers for them.
